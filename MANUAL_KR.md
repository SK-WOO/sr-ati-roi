# SR ATI ROI 계산기 — 사용 매뉴얼

**버전:** v1.12.2 · 2026-03-19
**URL:** https://sr-ati-roi.vercel.app/
**접근 권한:** `@seoulrobotics.org` Google Workspace 계정만 가능

---

## 이 도구는 뭐예요?

자동차 공장을 방문했을 때 고객이 이렇게 물어봤다고 상상해 보세요.
> *"당신네 로봇을 사면 돈이 얼마나 절약되고, 언제 본전이 나오나요?"*

이 도구가 생기기 전에는 엔지니어가 커다란 Excel 파일을 열고 수십 개의 숫자를 일일이 입력해서 30–60분이 지나야 답이 나왔습니다. 시간도 걸리고 실수도 잦았어요.

이 도구를 쓰면 **미팅 중에, 10분 안에** 답을 내고 고객에게 PDF 한 장을 바로 건네줄 수 있습니다.

---

## 시작 전 — 로그인

1. **https://sr-ati-roi.vercel.app/** 접속
2. **Google로 로그인** 클릭
3. `@seoulrobotics.org` 계정 선택
4. 요청이 오면 **허용** 클릭 (Google Sheets 읽기/쓰기 권한 — 팀 프리셋 공유용)
5. 로그인 완료. 팀원이 저장해 놓은 공장 설정이 자동으로 불러와집니다.

> `@seoulrobotics.org` 계정이 아니면 로그인이 차단됩니다.

---

## 전체 구조 — 어떻게 쓰나요?

앱은 **두 개 영역**으로 나뉩니다.

- **왼쪽 패널** — 숫자를 입력하는 5개 탭
- **오른쪽 패널** — 숫자가 바뀔 때마다 실시간으로 업데이트되는 결과

**탭은 왼쪽에서 오른쪽 순서로 채워야 합니다.** 각 탭의 결과가 다음 탭의 입력값으로 이어지거든요.

```
📦 생산 → 🚗 운송 → 👷 인력 → 🤖 SR Pricing → 🧮 가격 계산
```

### 불러온 프리셋 배지 (프리셋 로드 시 표시)

**어떻게 생겼나요:** `불러옴: Toyota · 일본 · Motomachi  [업데이트] [✕]`

**뜻:** 현재 저장된 프리셋 하나를 불러온 상태입니다. 화면의 모든 값이 그 프리셋에서 온 것입니다.

- **`업데이트` 버튼** — 현재 입력값으로 Google Sheets의 해당 프리셋을 덮어씁니다. 숫자를 수정한 뒤 새 버전으로 저장하고 싶을 때 사용.

## 왼쪽 패널 — 5개 탭

---

### 📦 탭 1: 생산 (Production)

**목적:** 공장의 운영 방식을 알려주세요. 연간 생산 목표, 근무 시간, SR이 담당할 비율을 입력합니다.

---

#### 국가 선택 (드롭다운)

**역할:** 공장이 위치한 국가를 선택합니다. 선택 즉시 해당 국가의 공휴일 수와 참고 연봉이 자동으로 채워집니다.

**아래에 나타나는 세 가지 정보 상자:**
- **공휴일 수** — 해당 국가의 연간 법정 공휴일. 실제 가동일에서 이 수가 빠집니다.
- **평균 연봉 ($)** — 자동으로 채워지는 참고 연봉. 실제 고객사 임금이 다르면 직접 수정하세요.
- **임금 인상률 (%)** — 연간 인건비 상승률. ROI 전망 계산에서 미래 절감액을 현실적으로 반영합니다.

---

#### 정규 근무일

**뜻:** 공휴일 제외 전, 연간 공장 가동일 수.

**일반값:** 250–260일

**왜 중요한가:** 가동일이 많을수록 인건비가 높고, SR 도입 효과가 커집니다.

---

#### 공휴일 수 *(아래 힌트 텍스트 있음)*

**뜻:** 연간 공휴일 수. 정규 근무일에서 빼서 실제 가동일을 계산합니다.

**힌트:** *정규 근무일에서 빼서 실제 조업 캘린더를 계산합니다.*

**팁:** 국가 선택 시 자동 입력되지만 수동 수정 가능합니다.

---

#### 교대당 정규 시간

**뜻:** 한 교대에 초과 근무 제외한 정규 근무 시간.

**일반값:** 8시간

---

#### 교대당 초과 근무 시간

**뜻:** 정규 시간 위에 추가로 일하는 시간.

**일반값:** 0–2시간. 초과 근무가 없으면 0으로 유지하세요.

---

#### 일일 교대 수

**뜻:** 하루에 몇 번 교대로 운영되는지.

**선택지:** 1교대, 2교대, 3교대.

**왜 중요한가:** 교대가 많을수록 총 운전기사 수가 늘어나고, SR이 대체해야 할 인원도 많아집니다.

---

#### 연간 CAPA (대)

**뜻:** 공장의 연간 차량 생산 목표 대수.

**일반값:** 10만–50만 대

**왜 중요한가:** 모든 계산의 출발점입니다. 시간당 생산량(UPH), 운전기사 수, 운송 횟수 모두 이 숫자에서 나옵니다.

---

#### 양품률 (Yield Rate) % *(아래 힌트 텍스트 있음)*

**뜻:** 예정된 총 가동 시간 중 실제로 생산에 쓰인 시간의 비율. 라인 정지, 고장 등 비가동 시간은 제외합니다.

**힌트:** *총 가동 예정 시간 대비 실제 가동 시간 (라인 정지 제외)*

**일반값:** 85–98%

**왜 중요한가:** 양품률 95%라면 5%는 비가동입니다. 도구가 이를 보상하기 위해 UPH를 더 높게 역산하고, 그에 따라 운송 횟수와 운전기사 수가 늘어납니다.

---

#### SR 커버리지 비율 % *(아래 힌트 텍스트 있음)*

**뜻:** SR이 대체할 유인 운송 비율.

**힌트:** *SR이 담당하는 CAPA 비율*

**예시:** 100% = SR이 모든 운전기사를 대체. 50% = SR이 절반 담당, 나머지 절반은 유인 유지.

**왜 중요한가:** 50% 커버리지면 인건비 절감도 절반입니다. 단계적 도입이나 부분 적용 시나리오를 시뮬레이션할 때 사용하세요.

---

#### 파란 결과 상자 (생산 탭 하단)

입력값에서 자동 계산된 숫자들입니다. 직접 입력하지 않아도 됩니다.

| 표시 항목 | 의미 |
|---|---|
| **실제 근무일** | 정규 가동일 − 공휴일 = 실제 조업일 수 |
| **역산 UPH** | CAPA를 달성하기 위해 필요한 시간당 생산량 |
| **일일 운송 횟수 (전체)** | 하루에 필요한 총 운송 횟수 |
| **일일 운송 횟수 (SR)** | SR이 담당하는 운송 횟수 |
| **SR CAPA** | SR이 처리하는 연간 생산량 |
| **총 연간 시간** | 연간 전체 교대 가동 시간 합계 |

---

### 🚗 탭 2: 운송 (Transport)

**목적:** 유인 운전기사 한 명이 한 번 왕복 운송하는 데 얼마나 시간이 걸리는지 분석합니다. 이 사이클 타임으로 공장에 필요한 총 운전기사 수를 계산합니다.

한 번의 완전한 왕복을 시간 순서로 나누면:

```
[출발 준비] → [운전] → [주차] → [도보 1] →
[헤드웨이 대기] → [귀환 탑승] → [도보 2] → [기타 대기]
```

---

#### 거리 (m)

**뜻:** 편도 운송 거리(미터). 부품을 배달하고 돌아오는 거리의 절반.

**예시:** 150m = 150m 운전해서 부품 전달 후 복귀.

---

#### 속도 (km/h)

**뜻:** 공장 내 평균 운행 속도.

**일반값:** 공장 내부에서는 10–30 km/h.

---

#### 출발 준비 시간 (분)

**뜻:** 운전 시작 전 준비 시간 — 차량 점검, 서류 작업 등.

---

#### 주차 시간 (분)

**뜻:** 목적지에서 주차하고 다시 출발할 때까지 소요 시간.

---

#### 도보 1 시간 (분)

**뜻:** 주차한 차에서 실제 배달 지점까지 걸어가는 시간.

---

#### 헤드웨이 / 택트 타임 (분)

**뜻:** 연속 운송 사이의 간격 — 생산 라인 리듬에 맞춰 얼마나 자주 운송해야 하는지.

**왜 중요한가:** 택트 타임이 8분인데 한 번 운송에 5분이 걸리면, 운전기사는 3분을 기다립니다. 운송 시간이 택트 타임보다 길면 운전기사를 더 투입해야 합니다.

---

#### 귀환 탑승 시간 (분)

**뜻:** 배달 후 출발 지점으로 돌아오기 위해 버스, 자전거, 도보 등을 이용하는 시간.

---

#### 도보 2 시간 (분)

**뜻:** 귀환 후 다시 차량까지 걸어가는 시간.

---

#### 기타 대기 시간 (분)

**뜻:** 위에 해당하지 않는 추가 고정 시간 — 안전 점검, 서명, 게이트 대기 등.

---

#### 파란 결과 상자 (운송 탭 하단)

| 표시 항목 | 의미 |
|---|---|
| **사이클 타임** | 완전한 왕복 1회에 걸리는 총 시간 |
| **교대당 운행 횟수** | 운전기사 1명이 1교대에 완주할 수 있는 운송 횟수 |
| **SR 운전기사 (전체)** | SR이 대체하는 전체 교대 합산 운전기사 수 |
| **유인 운전기사** | SR 도입 후 남아있는 유인 운전기사 수 |

---

### 👷 탭 3: 인력 (Workforce)

**목적:** 공장이 현재 유인 운전기사에게 얼마를 쓰고 있는지 계산합니다. SR이 이 비용보다 저렴하게 운영되어야 ROI가 나옵니다.

---

#### 고용 형태

세 가지 옵션:

- **정규직 (연봉)** — 위 국가 박스에 입력한 연봉 사용
- **계약직** — 정규직과 같지만 다른 복리후생 비율 적용
- **시급제** — 시간당 단가를 직접 입력

---

#### 시급 / 연봉

고용 형태에 따라:
- **시급 ($/시간)** — 시급제 선택 시 입력
- **연간 기본급 ($)** — 정규직 또는 계약직 선택 시 입력

---

#### 초과 근무 / 휴일 가산율 (%)

**뜻:** 초과 근무 또는 휴일 근무에 대해 정규 시급에 추가로 지급하는 비율.

**예시:** 50% → 초과 근무 시간은 정규 시급의 1.5배.

---

#### 복리후생 / 추가 비용율 (%)

**뜻:** 기본급 위에 얹히는 사회보험, 연금, 퇴직금, 복지 비용 비율.

**일반값:** 국가별로 20–35%.

**왜 중요한가:** 연봉 3만 달러짜리 직원이 복리후생 포함하면 실제 3만 8천 달러가 들 수 있습니다. SR이 이 총 비용보다 저렴해야 합니다.

---

#### 오른쪽 패널 — 인력 결과

입력값에서 자동 계산됩니다.

| 표시 항목 | 의미 |
|---|---|
| **1인당 비용** | 운전기사 1명에 드는 연간 총 비용 (모든 수당 포함) |
| **유효 시급** | 실제로 운전기사 한 시간 일하는 데 드는 비용 |
| **연간 인건비 기준선** | 모든 유인 운전기사의 연간 총 비용 (ROI 계산에 직접 반영) |
| **연간 잔여 인건비** | SR 도입 후 남아있는 유인 운전기사 비용 |

---

### 🤖 탭 4: SR Pricing

**목적:** SR ATI의 가격 구조를 설정하고 전체 투자 수익(ROI) 분석을 확인합니다.

이 탭은 두 부분으로 나뉩니다: **왼쪽 입력** (CAPEX, OPEX, 프로젝션 설정) / **오른쪽 결과** (ROI 차트, 민감도 매트릭스, 시나리오 비교).

---

#### CAPEX 섹션

CAPEX = SR ATI를 구매하고 설치하는 데 드는 일회성 비용.

도구는 4단계로 계산합니다:

```
1단계: 기준 = HW + (NRE × 난이도 계수) + 설치비 + 기타
2단계: 오버헤드 후 = 기준 × (1 + 오버헤드 %)
3단계: 마진 후 = 오버헤드 후 × (1 + 마진 %)
4단계: 최종 CAPEX = 마진 후 × (1 − 할인율 %)
```

---

##### 하드웨어 비용 (HW) `$`

**뜻:** 센서, 서버, 액세서리 등 하드웨어 구매 총 비용.

**팁:** 가격 계산 탭에서 **적용** 버튼을 누르면 자동 입력됩니다. 직접 입력도 가능합니다.

---

##### NRE `$`

**뜻:** 비반복 엔지니어링(Non-Recurring Engineering) 비용. 이 특정 공장에 맞게 시스템을 설계하고 지도화하고 구성하는 일회성 엔지니어링 비용.

**팁:** 가격 계산 탭에서 자동 입력됩니다.

---

##### 난이도 계수

**뜻:** NRE에 곱해지는 현장 복잡도 배율. 어려운 현장일수록 엔지니어링 시간이 더 필요합니다.

- **1.0** = 표준 현장, 특별한 어려움 없음
- **1.2** = 중간 복잡도 (기본값)
- **1.5** = 매우 복잡 (실외, 다층, 좁은 경로 등)

**팁:** 가격 계산 탭의 현장 난이도 설정에서 자동으로 반영됩니다.

---

##### 설치비 `$`

**뜻:** SR 시스템을 현장에 실제로 설치하고 구축하는 비용.

**기본값:** $84,000 (개발자 라이센스 — 가격 계산 탭 적용 시 자동 입력).

---

##### 기타 CAPEX `$`

**뜻:** 위에 해당하지 않는 추가 일회성 비용 (예: 토목 공사, 네트워크 인프라).

---

##### 오버헤드 %

**뜻:** 직접 비용 위에 얹히는 회사 오버헤드 — 내부 지원, 행정, 보험.

**기본값:** 15%

---

##### 마진 %

**뜻:** 고객 판매가에 포함되는 SR의 이익률.

---

##### 초도 공장 할인율 %

**뜻:** 신규 고객사와의 첫 계약을 성사시키기 위한 상업 할인.

**기본값:** 40%

---

#### OPEX 섹션

OPEX = 설치 후 SR 시스템을 운영하고 유지하는 연간 반복 비용.

두 가지 모드:

- **운송 횟수 방식** — 1회 운송당 비용으로 계산
- **면적 방식** — 현장 면적(m²) 기준으로 계산 *(기본값이며 닛산 Excel과 동일)*

**면적 방식의 구성 항목:**

| 항목 | 의미 |
|---|---|
| **HW 보증 비율 %** | HW 구매가 대비 연간 보증 비용 비율 |
| **현장 지원 ($/m²)** | 현장 기술자 방문, 교정, 유지보수 비용 |
| **SW 업데이트 ($/m²)** | 연간 소프트웨어 업데이트 비용 |
| **운영 라이센스 ($)** | 연간 소프트웨어 라이센스 비용 (가격 계산 탭 적용 시 자동 입력) |
| **오버홀 비율 %** | HW 가격 대비 전체 오버홀 비용 비율, 주기당 |
| **오버홀 주기 (년)** | 오버홀 빈도 (기본 5년 = 연간 10%) |

최종 OPEX에는 오버헤드 %와 마진 %가 추가로 적용됩니다.

---

#### OPEX 할인 일정

SR은 초기 몇 년간 OPEX 할인을 제공해서 고객의 ROI 초기 수치를 개선합니다.

- **1년차 할인율 %** — 첫 해에 얼마나 할인하는지 (기본 30%)
- **연간 감소 %** — 매년 할인율이 얼마씩 줄어드는지 (기본 3% → 2년차 27%, 3년차 24%...)

---

#### 프로젝션 설정

| 항목 | 의미 |
|---|---|
| **내용연수 (년)** | SR 하드웨어가 사용 가능한 기간. CAPEX를 이 기간으로 나눠서 연간 감가상각 계산. |
| **프로젝트 기간 (년)** | ROI 분석이 몇 년을 다루는지. |
| **SR 비용 증가율 %/년** | SR의 연간 OPEX가 매년 얼마씩 증가하는지 예상치. |

---

### 오른쪽 패널 — ROI 결과

앱 어디에서든 숫자를 바꾸면 이 패널이 즉시 업데이트됩니다.

---

#### KPI 카드 (상단 세 개 박스)

| 카드 | 의미 |
|---|---|
| **N년 순절감액** | 프로젝트 전 기간에 SR 비용을 모두 제외하고 남는 절감 금액 |
| **총 ROI %** | 순절감액 ÷ CAPEX. 100%라면 투자금을 회수하고 그만큼 더 절감한 것. |
| **최대 연간 절감** | 한 해에 최대로 가능한 절감액 (인건비 기준선 − SR OPEX) |

---

#### 연도별 ROI 테이블

프로젝트 각 연도의 현황을 표로 보여줍니다.

| 컬럼 | 의미 |
|---|---|
| **연도** | 1년차, 2년차... |
| **인건비 기준선** | 해당 연도에 유인 운전기사에게 지출될 비용 (임금 인상률 반영) |
| **잔여 유인 인건비** | SR 도입 후 남아있는 유인 운전기사 비용 |
| **SR OPEX** | 해당 연도 SR 운영 비용 (할인 일정 반영) |
| **SR 감가상각** | 해당 연도 CAPEX 감가 (CAPEX ÷ 내용연수) |
| **SR 합계** | 잔여 인건비 + SR OPEX + SR 감가상각 = 총 연간 SR 관련 비용 |
| **누적 절감액** | 지금까지 (인건비 기준선 − SR 합계)의 누적 합계 |
| **ROI** | 누적 절감액 ÷ CAPEX |

초록색 행 = 누적 절감액이 플러스로 전환된 연도 (SR이 수익을 내기 시작한 해).

**↓ CSV 버튼** — 이 표를 Excel에서 열 수 있는 스프레드시트 파일로 다운로드합니다.

---

#### 📊 민감도 분석 매트릭스

**뭐예요?** CAPEX와 인건비 변수를 각각 5개 수준으로 조합한 5×5 격자표. 25가지 시나리오의 ROI%를 한눈에 보여줍니다. 다시 입력할 필요 없음.

**두 개의 축:**
- **행 (왼쪽) = CAPEX 변화** — 하드웨어 비용이 더 비싸거나 저렴하면?
- **열 (위쪽) = 인건비 변화** — 고객사 임금이 더 높거나 낮으면?

**각 셀 = 그 조합의 ROI%.**

**색상 코딩:**
- 🟢 진초록 = ROI ≥ 50% (좋은 결과)
- 🟢 연초록 = ROI 0–50% (수익 발생, 본전 이상)
- 🔴 빨강 = ROI < 0% (이 시나리오에서는 손해)
- 🔵 파랑 = 기준 케이스 (현재 입력값)

**셀 위에 마우스를 올리면** 해당 시나리오의 손익분기점(BEP)과 누적 절감액이 표시됩니다.

**축 값 편집 방법:**
- 열 헤더(위쪽 행)나 행 헤더(왼쪽 열)의 숫자를 직접 클릭
- 원하는 숫자를 입력 — 양수, 음수, 어떤 값이든 가능
- 매트릭스가 즉시 재계산됨
- 기본값: −20%, −10%, 0%, +10%, +20%
- **예시:** CAPEX 행의 −20%를 −50%로 바꾸면 공격적 할인 시나리오를 바로 확인 가능

---

#### 💾 시나리오 A 저장 / B 저장 / 📊 비교

**역할:** 두 가지 설정을 나란히 비교할 수 있습니다.

**사용법:**
1. 첫 번째 조건을 설정합니다 (예: A 공장, SR 40% 커버리지)
2. **💾 시나리오 A 저장** 클릭 — 초록 ✓ 표시로 저장 확인
3. 입력값을 바꿉니다 (예: B 공장, 80% 커버리지, 더 높은 CAPEX)
4. **💾 시나리오 B 저장** 클릭
5. **📊 비교** 클릭 → 9개 핵심 지표를 나란히 보여주는 모달 표시

**언제 쓰나요:** 고객이 "일단 50%만 도입하면 어때요?"라고 물으면 — A로 50% 저장, 100%로 바꿔 B로 저장, 비교 화면을 보여주세요.

---

#### 분석 차트 (테이블 아래)

자동으로 업데이트되는 5가지 차트. 별도 입력 불필요.

| 차트 | 내용 |
|---|---|
| **CAPEX 구성 (파이)** | HW, NRE, 설치비, 기타, 오버헤드, 마진으로 나뉜 CAPEX 비율 |
| **연간 비용 구성 (파이)** | 1년차 기준: 잔여 인건비, SR OPEX, SR 감가상각, 절감된 인건비 비율 |
| **누적 절감액 추이 (면적 차트)** | 연도별 절감액 누적 곡선 — 0을 넘는 시점이 BEP |
| **ROI 레이더** | ROI%, BEP 속도, 인건비 절감%, OPEX 효율, 1년차 절감%, 커버리지% 6개 지표 |
| **연도별 순절감액 (막대)** | 연도별 순절감액을 막대 그래프로 |

---

### 🧮 탭 5: 가격 계산 (Pricing Calc)

**목적:** 닛산 가격 산정 Excel 로직을 그대로 구현합니다. 현장 면적과 하드웨어 수량을 입력하면 CAPEX와 OPEX가 자동 계산됩니다.

**마지막에 적용 버튼을 누르면 → 모든 값이 SR Pricing 탭으로 이동합니다.**

---

#### 현장(Site) 설정

"현장(Site)"은 SR이 운영될 공장 내 한 구역입니다. 도장 공장, 조립 라인, 주차장 등 여러 구역을 따로 추가할 수 있습니다.

**+ 현장 추가 버튼** — 새 현장 카드를 추가합니다.

**현장 이름** — 알아보기 쉬운 이름 입력 (예: "조립 라인 3").

**현장 유형 — 두 가지:**

| 유형 | 언제 쓰나 | 입력 항목 |
|---|---|---|
| **면적** | 넓은 바닥 면적 (가장 일반적) | 너비(m) × 높이(m) |
| **경로** | 정해진 이동 경로 (복도, 좁은 통로) | 경로 길이(m) |

현장 면적(m²)은 OPEX 계산에 직접 사용됩니다.

---

#### 난이도 계수 (DF) — 현장당 7개 슬라이더

각 슬라이더가 현장의 복잡도 점수에 더해집니다. 합산 DF는 해당 현장의 NRE에 곱해집니다 — 어려운 현장 = 더 많은 엔지니어링 = 높은 NRE.

| 항목 | 의미 |
|---|---|
| **실외 여부** | 경로가 외부에 있거나 조명·날씨 변화에 노출 |
| **경사** | 경사로 또는 다층 구조 이동 포함 |
| **도로 폭** | 좁거나 불규칙한 경로 |
| **노면** | 울퉁불퉁하거나 반사면, 젖은 바닥, 특수 마킹 |
| **복잡도** | 교차로 수, 교통량, 복잡한 동선 |
| **포장 상태** | 노면 품질이 낮거나 혼합 |
| **용량** | 매우 많은 차량이 동시에 이동 |

**🗑 현장 삭제 버튼** — 해당 현장을 삭제합니다. 취소 불가.

---

#### NRE 인력 구성

NRE(엔지니어링 비용)는 실제 투입 인원과 기간으로 계산됩니다.

```
기본 NRE = Σ (인원수 × 개월수 × 월 단가)  — 각 역할 합산
현장별 NRE = 기본 NRE × (이 현장 면적 ÷ 전체 면적) × 이 현장 DF
```

**테이블 컬럼:**

| 컬럼 | 입력 내용 |
|---|---|
| **역할명** | 직책 (예: Jr. Engineer, PM) |
| **월 단가 ($)** | 해당 역할의 1인 1개월 비용 |
| **HC** | 인원수 |
| **MM** | Man-month — 프로젝트에 투입하는 개월 수 |

**➕ 행 추가 버튼** — 새 역할을 추가합니다.

**🔄 기본값 초기화 버튼** — 닛산 Excel 기준값으로 되돌립니다:

| 역할 | 월 단가 | HC | MM |
|---|---|---|---|
| Jr. Engineer | $7,351 | 5명 | 8개월 |
| Mid Engineer | $9,905 | 3명 | 8개월 |
| Sr. Engineer | $11,607 | 2명 | 8개월 |

**🗑 (행 삭제)** — 해당 역할 행을 삭제합니다.

---

#### 하드웨어 구성

SR 하드웨어의 BOM(자재 명세서).

**기본 항목:** Lidar XT32, OT128, QT128, ZT128, Server, Accessories

각 항목:
- **단가 ($)** — 직접 편집 가능
- **수량** — 필요 대수
- **합계** = 단가 × 수량 (자동 계산)

**➕ 항목 추가 버튼** — 커스텀 하드웨어 항목 추가 (예: 새 센서 모델, 맞춤 마운트).

**🗑 (행 삭제)** — 해당 항목 삭제.

**HW 프리셋 버튼:**
- **💾 HW 구성 저장** — 현재 하드웨어 목록을 이름 붙여 브라우저에 저장
- **📂 HW 구성 불러오기** — 저장된 하드웨어 목록 로드 — 매번 같은 장비를 쓸 때 유용

---

#### 연간 처리량 (대/년)

**뜻:** SR이 담당하는 구역을 통과하는 연간 차량 수.

**왜 중요한가:** SW 라이센스 요금이 차량 볼륨에 따라 계단식으로 적용됩니다 (닛산 Excel 기준).

---

#### SW 라이센스 (OPEX 구성 요소)

자동 계산됩니다:

```
SW 라이센스 = 조정 면적 × 단위 단가 × 차량 중량 계수
```

차량 중량 계수는 연간 처리량에 따라 단계별로 올라갑니다. 계산된 값이 표시되며, 적용 버튼을 누르면 OPEX에 자동 반영됩니다.

---

#### SR Pricing 탭에 적용 (파란 버튼)

**역할:** 이 탭의 모든 계산값을 SR Pricing 탭 입력란으로 자동 이전합니다:

| SR Pricing 입력란 | 가격 계산 탭 출처 |
|---|---|
| CAPEX HW | 전체 하드웨어 (단가 × 수량) 합계 |
| CAPEX NRE | 전체 현장 NRE 합계 |
| CAPEX 설치비 | $84,000 고정 (Developer License) |
| OPEX 면적 (m²) | 전체 현장 면적 합계 |
| 운영 라이센스 | 위에서 계산된 SW 라이센스 |

적용 후 자동으로 SR Pricing 탭으로 전환됩니다.

## 프리셋 패널

헤더의 **🏭 프리셋** 버튼으로 엽니다.

팀 전체가 공유하는 공장 설정 라이브러리입니다. 프리셋을 저장하면 팀원이 즉시 볼 수 있습니다.

---

### 🔄 새로고침 버튼 (패널 우측 상단)

**역할:** 지금 당장 Google Sheets에서 최신 프리셋을 다시 불러옵니다.

**언제 쓰나요:** 동료가 방금 새 프리셋을 저장했는데 로그아웃 없이 바로 보고 싶을 때.

---

### 📂 JSON 가져오기 버튼 (패널 우측 상단)

**역할:** 컴퓨터에 저장된 JSON 파일(이전에 내보낸 프리셋 파일)을 불러옵니다.

**언제 쓰나요:** 동료가 이메일이나 메신저로 프리셋 파일을 공유해 준 경우.

---

### 각 프리셋 카드

저장된 프리셋 하나하나가 카드로 표시됩니다:

- **공장명** — 브랜드 · 국가 · 공장
- **작성자** — 저장한 사람
- **날짜** — 저장 일자
- **메모** — 추가 설명

**카드 버튼:**

| 버튼 | 역할 |
|---|---|
| **불러오기** | 이 프리셋의 값으로 현재 입력 전체를 교체 |
| **↓ 내보내기** | 이 프리셋을 JSON 파일로 컴퓨터에 다운로드 |
| **🗑 삭제** | Google Sheets에서 이 프리셋을 영구 삭제 — 주의! |

> 프리셋을 불러오면 헤더에 프리셋 이름과 **업데이트** 버튼이 나타납니다. 숫자를 수정한 뒤 **업데이트**를 누르면 같은 프리셋을 덮어씁니다.

팀 내부용. 오버헤드와 마진 포함 전체 비용 내역이 포함됩니다.

| **고객사 / OEM** | 고객 회사명 |

**추가 입력 항목:**

**견적 번호**는 PDF 생성 시 `QT-YYYYMMDD-HHMM` 형식으로 자동 생성됩니다.

**📥 견적서 다운로드 (PDF) 버튼** — 견적서 PDF를 생성해서 다운로드합니다.

**PDF 포함 내용:**

- OPEX 세부 내역
- ROI 하이라이트
- 거래 조건 (T&C)

**Drive 토큰이 만료된 경우:** 앱이 자동으로 재인증 요청을 보내고 재시도합니다. 로그아웃 불필요.

## 문제 해결

| 증상 | 가능한 원인 | 조치 |
|---|---|---|
| 로그인 버튼이 표시되지 않음 | 브라우저가 Google 로그인 스크립트를 차단 | 광고 차단기를 끄거나 다른 브라우저 사용 |
| "Sheets 로드 실패 (HTTP 401)" | Sheets 토큰 만료 | 로그아웃 후 재로그인 |
| "Sheets 로드 실패 (HTTP 403)" | API 권한 문제 | Workspace 관리자에게 문의 |
| 프리셋이 저장되지 않음 | 오프라인 또는 Sheets 불가 | 브라우저 로컬에는 저장됨, 연결 복구 후 재시도 |
| Drive 업로드 실패 | Drive API 권한 없음 | Workspace 관리자가 `drive` scope를 승인해야 함 |
| 숫자가 `—` 또는 음수로 표시됨 | 입력이 누락되거나 0 | 탭을 순서대로 채웠는지 확인, 특히 운송·인력 탭 |
| 입력창에 주황색 테두리 + ⚠ 표시 | 입력값이 허용 범위를 벗어남 | 힌트 텍스트의 최소/최대값을 확인하고 수정 |
