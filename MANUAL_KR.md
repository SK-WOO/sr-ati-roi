## 도구 개발 배경 및 목적

자동차 공장 고객은 ATI 도입 결정 전 반드시 이 질문을 합니다. *"투자 회수 기간은 얼마나 되고, 실제로 얼마를 절감할 수 있나요?"*

기존에는 이 질문에 답하기 위해 엔지니어가 닛산 가격 산정 Excel 파일을 열고 현장 면적, 하드웨어 수량, 인건비율, 할인 일정을 하나씩 채워야 했습니다. 고객 미팅 전후로 30–60분이 소요되고, 시간 압박 속에서 실수도 잦았습니다.

SR ATI ROI 계산기는 그 Excel 로직을 브라우저 기반으로 구현한 도구입니다. **고객 미팅 중 또는 미팅 전 10분 이내**에 구조화된 ROI 분석 결과와 바로 출력 가능한 PDF — 내부 보고서 또는 고객용 견적서 — 를 생성합니다.

| 결과물 | 설명 |
|---|---|

| CAPEX 견적 | HW + NRE + 설치비, 오버헤드·마진·할인 적용 후 최종 금액 |
| 연간 OPEX | HW 보증, 현장 지원, SW 업데이트, 오버홀, 운영 라이센스 |

## 탭 순서 및 워크플로

앱은 5개 탭으로 구성됩니다. **왼쪽에서 오른쪽 순서로** 입력합니다.

### 1. 📦 Production (생산)

고객 공장의 생산 파라미터를 입력합니다.

| 입력 항목 | 설명 |
|---|---|
| 연간 CAPA | 공장 연간 차량 생산 목표 |
| 수율 % | 불량 제외 실질 생산율 |
| 일일 교대 수 | 운영 교대 횟수 |
| 연간 가동일 | 휴일 제외 조업일 |
| 교대당 시간 | 정규 시간 + 초과 근무 시간 |
| SR 커버리지 % | ATI로 대체되는 유인 운행 비율 |

**출력:** 시간당 생산량(UPH), 필요 SR 대수, 교대당 필요 운전기사 수

---

### 2. 🚗 Transport (운송)

**8단계 유인 운행 사이클** 시간을 분석하여 공장이 실제 필요한 운전기사 수를 계산합니다.

```
출발 준비 → 운전 → 주차 → 도보 1 → 헤드웨이 대기 → 귀환 탑승 → 도보 2 → 기타 대기
```

| 입력 항목 | 설명 |
|---|---|
| 거리 (m) | 편도 이동 거리 |
| 속도 (km/h) | 평균 운행 속도 |
| 출발 준비 / 주차 / 도보 / 기타 | 각 단계별 고정 시간 (분) |
| 헤드웨이 | 택트 타임 (분) |
| 귀환 탑승 시간 | 복귀 탑승 시간 (분) |

**출력:** 사이클 시간, 교대당 운행 횟수, 총 운전기사 수 → 인건비 기준선 입력값

---

### 3. 👷 Workforce (인력)

SR ATI가 대체해야 할 **연간 인건비 기준선**을 계산합니다.

| 입력 항목 | 설명 |
|---|---|
| 국가 | 국가별 임금 데이터 내장 |
| 임금 수준 | 해당 국가의 하/중/상 |
| 고용 형태 | 정규직(연봉) 또는 시급 |
| 정규/초과/휴일 근무 시간 | 가산율 포함 실제 일정 |
| 복리후생 비율 % | 사회보험, 퇴직금, 복지 오버헤드 |
| 임금 인상률 % | 프로젝트 기간 연평균 임금 상승률 |

**출력:** 1인당 연간 비용, 총 연간 인건비 기준선 (ROI 계산에 직접 입력됨)

### 4. 🤖 SR Pricing

SR ATI 비용 구조를 설정하고 전체 ROI 분석을 검토합니다.

#### CAPEX 계산

```
CAPEX 기준  = HW + NRE × 난이도 계수 + 설치비 + 기타
오버헤드 후 = CAPEX 기준 × (1 + 오버헤드 %)
마진 후     = 오버헤드 후 × (1 + 마진 %)
최종 CAPEX  = 마진 후 × (1 − 할인율 %)
```

| 항목 | 설명 |
|---|---|
| HW | 하드웨어 총 비용 |
| NRE | 비반복 엔지니어링 비용 |
| 난이도 계수 | NRE에 적용되는 현장 복잡도 배율 (1.0 = 표준) |
| 설치비 | Pricing Calc 적용 시 $84,000 (Dev License) 자동 설정 |
| 오버헤드 % | 기본값 15% |
| 마진 % | 자유 설정 |
| 할인율 % | 초도 공장 또는 상업 할인 |

#### OPEX 계산 (면적 방식)

```
직접 OPEX = HW 보증 비용 (HW의 %)
          + 현장 지원 ($/m²)
          + SW 업데이트 ($/m²)
          + 운영 라이센스 ($)
          + 오버홀 (HW의 % ÷ 주기 연수)

연간 OPEX = 직접 OPEX × (1 + 오버헤드%) × (1 + 마진%)
```

> **참고:** 오버홀 기본값은 **5년 주기당 HW 비용의 50%** (= 연간 10%)로, 닛산 Excel 벤치마크 값입니다.

> **운영 라이센스**는 Pricing Calc 탭에서 *적용* 버튼을 누르면 자동 입력되며, 직접 수정도 가능합니다.

#### OPEX 할인 일정

1년차 할인율(기본 30%)에서 매년 설정한 step(기본 3%)씩 감소합니다. SR 상업 협약의 표준 램프다운 구조를 반영합니다.

#### ROI 분석 패널

연도별 누적 결과를 테이블과 차트로 표시합니다.

| 항목 | 설명 |
|---|---|
| 인건비 기준선 | 인플레이션 반영 연간 인건비 |
| 잔여 유인 인건비 | SR 도입 후 남아있는 유인 운전기사 비용 |
| SR OPEX | 연간 운영 비용 (할인 일정 적용) |
| SR 감가상각 | CAPEX ÷ 내용연수 |
| 누적 절감액 | (누적 인건비) − (누적 SR 비용) 추이 |

**KPI 요약 카드:** 전체 기간 순절감액 · 총 ROI % · 최대 연간 절감액

#### 📊 민감도 분석 *(v1.10.0)*

데이터를 다시 입력하지 않고 5가지 시나리오를 즉시 계산합니다.

| 시나리오 | 변경 내용 |
|---|---|
| CAPEX −20% | 대량 구매 할인 등으로 HW/NRE 비용 감소 |
| CAPEX −10% | 중간 수준 할인 |
| ▶ 기준 | 현재 입력값 |
| 인건비 +10% | 고객 인건비가 가정보다 빠르게 증가 |
| 인건비 +20% | 공격적 인건비 상승 시나리오 |

각 행에 손익분기점, ROI%, 누적 절감액이 표시됩니다. 협상 자리에서 결과 범위를 보여줄 때 활용하세요.

#### 💾 시나리오 A/B 비교 *(v1.10.0)*

1. 첫 번째 조건 설정 (예: A 공장, 40% 커버리지) → **시나리오 A 저장** 클릭
2. 입력값 변경 (예: B 공장, 60% 커버리지) → **시나리오 B 저장** 클릭
3. **비교** 클릭 → 9개 핵심 지표 나란히 비교 모달

두 공장, 두 가지 할인율, 또는 두 가지 커버리지 시나리오를 한 자리에서 비교할 때 유용합니다.

### 5. 🧮 Pricing Calc (가격 계산)

**닛산 가격 산정 Excel 로직**을 그대로 구현합니다. 현장 면적과 하드웨어 수량을 입력하면 CAPEX와 OPEX가 자동 계산됩니다.

#### 현장 (Site) 설정

각 현장은 두 가지 방식으로 설정 가능합니다.
- **면적 방식:** 너비(m) × 높이(m) = 총 m²
- **경로 방식:** 경로 길이(m)

**난이도 계수 (DF)** — 현장별 7개 조정 항목:

| 항목 | 의미 |
|---|---|
| 실외 여부 | 날씨·조명 변화 노출 |
| 경사 | 램프 또는 다층 구조 |
| 도로 폭 | 좁거나 불규칙한 경로 |
| 노면 | 불균일·반사·습한 표면 |
| 복잡도 | 교차로 수, 교통 밀도 |
| 포장 상태 | 노면 품질 |
| 용량 | 고처리량 차선 |

DF 합산값이 해당 현장 NRE에 곱해집니다.

#### NRE 인력 구성 *(v1.10.0)*

NRE는 이제 고정 단가가 아닌 **실제 인력 투입 기준**으로 계산됩니다.

```
Base Labor NRE = Σ (HC × MM × 월 단가)   — 각 역할별 합산
현장별 NRE    = Base Labor NRE × (현장 m² / 전체 m²) × 현장 DF
```

기본값은 닛산 Excel 기준 인건비:

| 역할 | 월 단가 | 기본 HC | 기본 MM |
|---|---|---|---|
| Jr. Engineer | $7,351/월 | 5명 | 8개월 |
| Mid Engineer | $9,905/월 | 3명 | 8개월 |
| Sr. Engineer | $11,607/월 | 2명 | 8개월 |

**+ 행 추가** 버튼으로 어떤 역할이든 (예: PM, 통합 전문가) 자유롭게 추가 가능합니다. 역할명, 월 단가, 인원 수, 기간을 직접 편집하세요. **기본값 초기화** 버튼으로 표준 구성으로 되돌릴 수 있습니다.

#### 하드웨어 구성

기본 라인업: Lidar XT32, OT128, QT128, ZT128, Server, Accessories — 단가 편집 가능. 커스텀 항목 추가 및 HW 프리셋 저장/불러오기 지원.

#### SW 라이센스 (OPEX)

```
SW 라이센스 = 조정 면적 × 단위 단가 × 차량 중량 계수
```
차량 중량 계수는 연간 처리량(대수)에 따라 계단식 적용됩니다 (닛산 Excel 기준).

#### SR Pricing 탭에 적용

**SR Pricing 탭에 적용** 버튼을 누르면 모든 계산값이 자동으로 이전됩니다.

| SR Pricing 항목 | 출처 |
|---|---|
| CAPEX HW | HW BOM 합산 |
| CAPEX NRE | 현장별 분배된 NRE 합산 |
| CAPEX 설치비 | $84,000 (Dev License 고정) |
| OPEX 면적 | 전체 현장 면적 (m²) |
| 운영 라이센스 | SW 라이센스 자동 계산값 |

적용 후 SR Pricing 탭으로 자동 전환됩니다.

## PDF 보고서

헤더 우측의 **📄 Report** 버튼을 클릭합니다.

### 내부 보고서

경영진 보고 및 내부 검토용.

**파일명:** `SR-ROI-Report_[고객사]_[작성자]_[YYYYMMDD].pdf`

**포함 내용:**
- 프로젝트 정보 (고객사, 작성자, 부서, 날짜)
- 생산 및 운송 KPI
- 인력 기준선
- CAPEX 세부 내역 (4단계)
- OPEX 및 ROI 요약
- 연도별 ROI 전체 테이블

### 고객 견적서

고객에게 직접 전달하는 공식 견적서.

**견적 번호:** 자동 생성 `QT-YYYYMMDD-HHMM`
**파일명:** `SR-Quotation_[고객사]_[영업담당]_[YYYYMMDD].pdf`

**포함 내용:**
- 커버 페이지 (견적 번호, 유효기간, SR 브랜딩)
- 하드웨어 BOM
- 단계별 CAPEX 산출
- OPEX 세부 내역
- ROI 하이라이트
- 거래 조건 (T&C)

## 팀 프리셋 (Google Sheets)

팀 프리셋을 사용하면 전체 입력값 세트를 고객/공장 이름으로 저장하고 팀 전체와 즉시 공유할 수 있습니다.

- 로그인 시 Google Sheets에서 자동으로 프리셋을 불러옵니다
- 헤더의 **🏭 Presets** 버튼으로 프리셋 열람·불러오기·삭제 가능
- **💾 Update** 버튼으로 현재 입력값을 불러온 프리셋에 덮어씁니다
- **🔄 새로고침** 으로 로그아웃 없이 최신 프리셋을 즉시 동기화
- 오프라인 시 localStorage에 저장된 프리셋을 대체로 사용합니다

**v1.10.0:** NRE 인력 구성과 운영 라이센스가 프리셋 저장에 포함되어, Pricing Calc 전체 설정이 팀과 공유됩니다.
